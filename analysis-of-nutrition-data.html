Analyzing the nutrition content of common foods¶

Table of Contents

Motivation¶

Federal nutrition programs have been a crucial initiative over the years, and a careful, impartial third party analysis to verify the sanctity of their recommendations is long overdue. An analysis by the CDC earlier this year revealed that the obesity rate in our country has hiked up to 42.4%, a dangerous new peak for the United States. Clearly, something in the system is not working. Is the government pushing for the right foods to be consumed?

Therefore, some important questions that one might want to answer with data science techniques are:

  1. What other nutrients is high protein content correlated with?
  2. What kinds of clusters of food exist? Can we identify some of these clusters as representing groups of "healthy" or "unhealthy" foods? If so, are there any surprising foods in either cluster?
  3. What can one do to reduce cholesterol and fat while keeping protein high in ones diet?

We can attempt to answer these questions by analyzing nutrition information of foods whose labels have been uploaded by public users to USDA's database https://data.nal.usda.gov/dataset/usda-branded-food-products-database. For further reading and understanding of the data, refer to the documentation which can be found in the previous link.

The specific dataset used in this project is available at https://data.nal.usda.gov/dataset/usda-branded-food-products-database/resource/c929dc84-1516-4ac7-bbb8-c0c191ca8cec.

Sanitizing data¶

This dataset is too large to do any kind of analysis in a reasonable amount of time (the nutrients table has over 3 million rows!), and contains many foods with little nutrition content. Let's isolate the first 100 foods in the dataset for which there is a significant amount of nutrients in the categories we want to analyze:

import numpy as np
seed = 10
np.random.seed(seed)

# Make a random choice which has not already been made
def make_random_choice(size, choices_already_made):
    
    # If we've already made all possible choices, we would get stuck in an infinite while loop so we must abort
    if len(choices_already_made) == size:
        print("No more choices left!")
        exit()
    
    # Keep picking until we hit something fresh
    attempts = 1
    choice = np.random.randint(low = 0, high = size)
    while choice in choices_already_made:
        choice = np.random.randint(low = 0, high = size)
        attempts = attempts + 1
    choices_already_made.append(choice)
    return choice, choices_already_made, attempts

Process Data¶

In order to analyze the nutrition content of these foods and compare them against each other, we will need to create a new database which organizes the data into one row for each food, with all of its nutrition information alongside it

%%timeit

Speeding up our processing¶

This is very slow! Let's try to speed it up. Pandas comes with a built-in apply() method which works much faster than iteration using iterrows(), which is incidentally already about twice as fast as manual iteration using iloc[i++]

Further Reading¶

This resource https://towardsdatascience.com/how-to-make-your-pandas-loop-71-803-times-faster-805030df4f06 explains how this speedup takes place and suggests even faster methods such as Numpy vectorization. However, it's not clear how to extend the Numpy vectorization method to work on two tables instead of just one. So for now, we'll stick with this.

Clustering data with k-means¶

In order to answer our questions from earlier, we use k-means to cluster the data into groups. This algorithm works by choosing k random points and organizing data based on their distance to those points, then resetting the k means to be the centroids of the groups and iterating this process until convergence.

To implement k-means on our data, we will make use of sklearn's built in unsupervised learning methods.
