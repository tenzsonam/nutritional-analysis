Analyzing the nutrition content of common foods¶

Siddhartha Harmalkar and Tenzin Sonam¶

CMSC320 Summer 2020 Final Tutorial¶

Federal nutrition programs have been a crucial initiative over the years, and a careful, impartial third party analysis to verify the sanctity of their recommendations is long overdue. An analysis by the CDC earlier this year revealed that the obesity rate in our country has hiked up to 42.4%, a dangerous new peak for the United States. Clearly, something in the system is not working. Is the government pushing for the right foods to be consumed?

Therefore, some important questions that one might want to answer with data science techniques are:

  1. What other nutrients is high protein content correlated with?
  2. What kinds of clusters of food exist? Can we identify some of these clusters as representing groups of "healthy" or "unhealthy" foods? If so, are there any surprising foods in either cluster?
  3. What can one do to reduce cholesterol and fat while keeping protein high in ones diet?

In this project, we attempt to answer these questions by analyzing nutrition information of foods whose labels have been uploaded by public users to USDA's database https://data.nal.usda.gov/dataset/usda-branded-food-products-database.

The specific dataset used in this project is available at https://data.nal.usda.gov/dataset/usda-branded-food-products-database/resource/c929dc84-1516-4ac7-bbb8-c0c191ca8cec.

  • LCSG Calculated from a 'greater than' value per serving size measure

Shortening data¶

This dataset is just too big (the nutrients table has over 3 million entries!) so let's only look at the first 1,000 foods in the dataset for which there is a significant amount of nutrients in the categories we want to analyze:

In [4]:
n = 100
nutrients_to_analyze = ['Energy', 'Protein', 'Total lipid (fat)', 'Cholesterol']

# One way to select foods to analyze: Grab the first n foods which have nonzero nutrient information in the relevant columns
def get_good_foods():
    counter = 0
    i = 0
    good_foods = pandas.DataFrame(columns=products.columns)
    while(counter < n):
        product = products.iloc[i]

        # Check if all desired nutrients are present in this product
        for nutrient in nutrients_to_analyze:
            good = True
            nutrient_search = nutrients[(nutrients.NDB_No == product.NDB_Number) & (nutrients.Nutrient_name == nutrient)]
            if nutrient_search.empty or nutrient_search.Output_value.iloc[0] > 0:
                good = False
                break

        # If so, add it to our smaller database
        if good:
            good_foods.append(product)
            counter = counter + 1

        # Keep looking
        i = i + 1
        print(i, counter)
    return good_foods

Clustering data with k-means¶

In order to answer our questions from earlier, we use k-means to cluster the data into groups:

In [25]:
from sklearn.preprocessing import StandardScaler, MinMaxScaler
from sklearn.mixture import GaussianMixture

# k - means
k = 4
model = GaussianMixture(n_components=k,
                        covariance_type="full",
                        random_state=1,
                        n_init=1,
                        max_iter=200,
                        init_params="kmeans")
X_train = processed_data[desired_columns].values
model.fit(X_train)
print("Model converged: " + str(model.converged_))
processed_data["cluster"] = model.predict(X_train)

Model converged: True

To check that our clustering has produced useful results, we can look at the level of correlation between cluster means. This gives us a measure of whether our clusters are producing groups of data with significant differences between them or not. Large correlation indicates that our groups have similar content, while low correlation indicates that our groups differ enough to be considered separate entities:
